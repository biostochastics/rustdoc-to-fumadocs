While rustdoc-to-fumadocs includes security measures such as:

- Input size limits (100MB maximum)
- Path sanitization to prevent directory traversal
- Output path validation
- Recursion limits to prevent stack overflow

These measures do not guarantee complete security. Users should:

- Review generated file paths before writing to disk
- Process rustdoc JSON from trusted sources
- Run in sandboxed environments when processing untrusted input
- Use `--dry-run` mode to preview output before writing
